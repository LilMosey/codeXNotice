# CodeX Notice 第 8 阶段计划：后台周期扫描

## 目标

让桌面 App 打开后持续监听当前机器上的 Codex 桌面任务完成情况，而不是只在启动瞬间扫描一次。当前 MVP 仍只发送 macOS 本地通知；钉钉、延迟合并通知先不实现。

## 范围

- Codex 状态目录不存在时视为空扫描，不影响 App 启动。
- 主进程启动后台线程，按固定间隔重复扫描和发送本地通知。
- 每一轮扫描使用当前时间重新构造运行配置。
- 保留现有规则、任务去重、事件记录和本地通知逻辑。

## 不做

- 不接入钉钉。
- 不实现非通知时间段后的合并补发。
- 不做完整系统托盘或开机自启动。

## 验证

- 为缺失 Codex 状态目录补测试。
- 运行 Rust 单元测试。
- 运行前端构建。
- 运行 Tauri App 打包。
